#  Task Manager QA Portfolio

This repository contains **comprehensive QA documentation and artifacts** for the **Task Manager application**, developed using **vibe coding (AI-assisted) tools like Bolt and Qoder**. The repository demonstrates the end-to-end QA process, from requirements gathering to test cases, bug reporting, and UX/UI review.

---

##  Project Overview

The Task Manager application is designed to help users manage tasks efficiently, including creating, editing, and deleting tasks, assigning priorities, and handling due dates. This QA portfolio captures **all manual testing activities and documentation** related to the app.

Purpose of this repository:

- Showcase **structured QA documentation**  
- Provide **detailed test cases** with expected results  
- Track **bugs and defects** with evidence screenshots  
- Maintain **version history and requirement artifacts**  


---

##  QA Documentation Details

### 1️⃣ Functional Requirement Document (FRD)
- Captures all app functionality in detail  
- Defines fields, forms, and validation rules  
- Provides a clear reference for testing

### 2️⃣ Product Requirement Document (PRD)
- Includes features and expected behaviors  
- Aligns development output with business needs

### 3️⃣ Test Cases
- Includes **positive, negative, and edge cases**  
- Covers task creation, editing, deletion, bulk actions, and validations  
- Each test case includes:
  - Precondition
  - Steps
  - Expected Result
  - Actual Result

### 4️⃣ Bug Reports
- Captures **all issues discovered during testing**  
- Includes severity, status (Open / Fixed / Verified)  
- Evidence attached via screenshots

### 5️⃣ UX/UI Images
- Screenshots of app screens  
- Helps visualize flows and design elements
